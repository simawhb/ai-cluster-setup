# 系统盘更换指南

## 方法一：磁盘克隆（推荐）

最简单的方式，直接把旧盘内容完整复制到新盘。

### 准备工作
1. 新固态硬盘（建议至少 256GB）
2. 硬盘盒或 SATA 转 USB 线（用于连接新盘）
3. 克隆软件

### 推荐软件
- **免费方案**：[Macrium Reflect Free](https://www.macrium.com/reflectfree) 或 [Clonezilla](https://clonezilla.org/)
- **付费方案**：[Acronis True Image](https://www.acronis.com/)（部分SSD品牌自带）

### 操作步骤（以 Macrium Reflect 为例）

1. **连接新盘**
   - 用硬盘盒把新SSD通过USB连接到电脑
   - 确认新盘被识别（磁盘管理中能看到）

2. **安装 Macrium Reflect**
   - 下载安装免费版
   - 启动后会看到所有磁盘

3. **执行克隆**
   - 选择源盘（当前C盘，112GB）
   - 点击"Clone this disk"
   - 选择目标盘（新SSD）
   - 调整分区大小（可选，建议把所有空间给C盘）
   - 点击"Finish" → "OK"开始克隆

4. **等待完成**
   - 112GB 数据约需 30-60 分钟（取决于USB速度）
   - 不要中断过程

5. **更换硬盘**
   - 关机
   - 拆下旧硬盘
   - 装上新硬盘
   - 开机测试

### 注意事项
- 克隆前关闭所有程序
- 确保新盘容量 ≥ 旧盘已用空间
- 克隆后旧盘可作为数据盘使用

## 方法二：全新安装 + 迁移

如果想要一个干净的系统，可以全新安装Windows再迁移数据。

### 准备工作
1. Windows 安装U盘（8GB以上）
2. 新固态硬盘
3. 备份重要数据

### 操作步骤

1. **制作安装U盘**
   - 下载 [Windows Media Creation Tool](https://www.microsoft.com/software-download/windows10)
   - 运行工具，选择"为另一台电脑创建安装介质"
   - 选择U盘，等待完成

2. **备份数据**
   ```
   需要备份的目录：
   - C:\Users\whb\Documents（文档）
   - C:\Users\whb\Desktop（桌面）
   - C:\Users\whb\Downloads（下载）
   - C:\Users\whb\AppData\Local\（部分应用数据）
   - 浏览器书签（导出为HTML）
   - 重要软件的配置文件
   ```

3. **安装新硬盘并安装系统**
   - 关机，拆下旧硬盘，装上新硬盘
   - 插入U盘，开机
   - 按 F2/Del 进入BIOS，设置U盘启动
   - 按提示安装Windows

4. **安装驱动和软件**
   - 安装主板、显卡、网卡驱动
   - 安装常用软件
   - 恢复备份的数据

5. **旧硬盘处理**
   - 旧硬盘可装入硬盘盒作为移动硬盘
   - 或安装到其他SATA口作为数据盘

## 方法三：使用 Windows 系统映像

Windows 自带的备份功能，但不如克隆灵活。

### 操作步骤

1. **创建系统映像**
   - 控制面板 → 备份和还原(Windows 7)
   - 点击"创建系统映像"
   - 选择备份位置（外部硬盘或网络位置）
   - 开始备份

2. **恢复到新盘**
   - 用Windows安装U盘启动
   - 选择"修复计算机"
   - 选择"系统映像恢复"
   - 选择之前创建的映像

## 针对你的具体情况

### 当前配置
- C盘：112GB（已用77GB，可用36GB）
- D盘：915GB（已用614GB，可用302GB）

### 建议方案
1. **推荐**：磁盘克隆（方法一）
   - 最简单，不丢失任何数据和设置
   - 所有软件、配置、环境变量都保留

2. **新盘容量建议**
   - 最小：256GB（够用但紧张）
   - 推荐：512GB（舒适）
   - 理想：1TB（充裕）

3. **旧盘利用**
   - 旧112GB SSD可装入硬盘盒
   - 作为D盘备份盘或移动存储

### 操作后检查清单
- [ ] 系统能正常启动
- [ ] 网络连接正常
- [ ] 所有软件能运行
- [ ] D盘数据可访问
- [ ] 环境变量正确（Python、Node.js路径）
- [ ] SSH密钥仍在（C:\Users\whb\.ssh\）
- [ ] 集群脚本能运行

## 常见问题

### Q: 克隆后新盘无法启动？
A: 进BIOS检查启动顺序，确保新盘是第一启动项。如果是UEFI模式，确保EFI分区正确。

### Q: 新盘比旧盘小怎么办？
A: 克隆前先清理数据，确保已用空间小于新盘容量。可以先删除大型软件，克隆后重新安装。

### Q: 克隆后盘符错乱？
A: 磁盘管理中手动更改盘符。确保C盘是系统盘，D盘是数据盘。

### Q: 旧盘数据还能用吗？
A: 可以。旧盘装入硬盘盒后，所有数据都在，可以直接访问。
